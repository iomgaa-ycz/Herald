# 数据分析总结与建模建议

## 1. 数据特征总结

### 1.1 核心特征
- **数据规模**：训练集 800,000 样本，测试集 100,000 样本（比例 8:1，适合深度学习和集成方法）
- **特征维度**：31 个特征，中等维度
- **特征类型分布**：
  - 连续型：16 个（已归一化，f_00~f_06, f_19~f_26, f_28）
  - 离散整数：14 个（小整数 0~9，f_07~f_18, f_29, f_30）
  - 高基数类别：1 个（f_27，670,919 个唯一值，几乎每条记录唯一）

### 1.2 数据质量
- **缺失值**：无缺失值 ✅
- **目标分布**：平衡（0类 51.4%，1类 48.6%），无需特殊处理类别不平衡
- **异常值**：连续特征已归一化，初步检查无明显异常
- **数据完整性**：测试集 100,000 样本需全部提交预测

---

## 2. 关键发现

### 2.1 特征交互是核心
竞赛描述明确强调 **"feature interactions"**，意味着：
- 单个特征预测能力可能有限
- 特征组合（如 f_00 × f_19，或 f_07 + f_08）可能包含关键信号
- 需要模型具备捕获高阶交互的能力

### 2.2 f_27 特征的特殊性
- **670,919 个唯一值** → 远超训练集样本数（800,000）
- 可能是某种唯一标识符或时间戳编码，直接 one-hot 编码不可行
- 建议处理方式：
  - 方案 1：Target Encoding（将类别映射为目标变量均值）
  - 方案 2：频率编码（统计出现次数）
  - 方案 3：直接丢弃（如果验证集上无贡献）

### 2.3 离散整数特征的潜力
- 14 个特征取值范围 0~9，可能是传感器读数或离散化后的测量值
- 可以同时作为数值特征（保留顺序信息）和类别特征（one-hot 捕获非线性）

---

## 3. 建模建议

### 3.1 推荐的建模方法

**优先级 1：基于树的集成模型**
- **LightGBM / XGBoost / CatBoost**
  - ✅ 原生支持类别特征（CatBoost 对 f_27 处理较好）
  - ✅ 通过树分裂自动捕获特征交互
  - ✅ 对归一化不敏感（适配当前数据）
  - ✅ 训练速度快，80万样本可接受

**优先级 2：深度学习（需谨慎）**
- **TabNet / FT-Transformer**
  - ⚠️ 需要充分的特征交互工程（手动构造二阶、三阶交互特征）
  - ⚠️ 训练时间长，超参数敏感
  - ⚠️ 对 f_27 需要额外 embedding 层

**优先级 3：线性模型（baseline）**
- **Logistic Regression + Polynomial Features**
  - 仅作为快速 baseline，预期 AUC 较低

### 3.2 特征工程方向

**阶段 1：基础处理**
1. f_27 处理：优先尝试 **Target Encoding**（需 5-fold CV 防止过拟合）
2. 离散整数特征：同时保留数值形式和 one-hot 形式

**阶段 2：交互特征**（关键！）
1. **二阶交互**：
   - 连续 × 连续：如 `f_00 * f_19`, `f_01 * f_20`（优先高相关组合）
   - 连续 × 离散：如 `f_00 * f_07`
   - 离散 × 离散：如 `f_07 + 10*f_08`（组合编码）

2. **聚合特征**：
   - 连续特征统计：`mean(f_00~f_06)`, `std(f_19~f_26)`
   - 离散特征计数：`sum(f_07~f_18 > 5)`

3. **特征选择**：
   - 基于 SHAP / Permutation Importance 筛选有效交互特征
   - 控制特征数量（避免过拟合）

### 3.3 评估指标细节

- **主指标**：AUC（ROC曲线下面积）
  - 对分类阈值不敏感，适合不平衡数据
  - 关注预测概率的排序能力，而非绝对值
- **验证策略**：5-fold Stratified CV（保持目标分布一致）
- **leaderboard 策略**：
  - Public LB（30%）可能与 Private LB（70%）分布不同
  - 避免过拟合 Public LB，以本地 CV 分数为准

### 3.4 预期难度与风险

**难度**：中等偏高
- ✅ 数据质量好，无缺失值
- ⚠️ 特征交互需要大量实验
- ⚠️ f_27 高基数特征处理复杂

**风险**：
1. **过拟合 f_27**：训练集唯一值过多，可能导致 Target Encoding 泄露
2. **特征爆炸**：交互特征组合数 = C(31,2) = 465 个，需筛选
3. **计算资源**：80万样本 + 大量特征，需要 16GB+ 内存

## 4. 模板选择理由

### 4.1 为什么选择 `tabular` 模板

- **数据类型匹配**：标准表格数据，非图像/文本/时序
- **任务类型匹配**：二分类 + AUC 评估，tabular 模板已内置最佳实践
- **工具链成熟**：sklearn + LightGBM + optuna 组合经过验证

**❌ 不选择 `generic` 的理由**：
- 需要从零实现特征工程管道
- 缺少表格数据的优化逻辑（如类别编码、交叉验证策略）

### 4.2 各 slot 的初步实现方向

| Slot | 功能 | 初步方向 |
|------|------|----------|
| `feature_engineering` | 特征构造 | 1. f_27 Target Encoding<br>2. 二阶交互特征（连续×连续优先）<br>3. 聚合统计特征 |
| `model_selection` | 模型选择 | LightGBM（主力） + XGBoost（对比）<br>参数：`learning_rate=0.05`, `max_depth=6`, `num_leaves=31` |
| `hyperparameter_tuning` | 超参数优化 | Optuna 贝叶斯优化，100次迭代<br>优化目标：5-fold CV AUC |
| `prediction` | 预测生成 | 软投票（LightGBM + XGBoost 概率平均）<br>输出格式：`{id, target}` |
