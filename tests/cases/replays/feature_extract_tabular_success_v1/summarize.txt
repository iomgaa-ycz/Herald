## 数据特征总结

| 维度 | 值 |
|------|-----|
| 训练集 | 800,000 × 33（含 id + target） |
| 测试集 | 100,000 × 32（含 id，无 target） |
| 任务 | 二分类，metric = AUC (maximize) |
| 缺失值 | **无** |
| 目标分布 | 0: 50.95% / 1: 49.05%，**基本平衡** |

**特征结构（三组 + 特殊列）：**

| 组别 | 列 | 类型 | 特点 |
|------|----|------|------|
| A-连续标准化 | f_00~f_06 (7列) | float64 | mean≈0, std≈1，已标准化 |
| B-离散计数 | f_07~f_18 (12列) | int64 | 0~13，正偏，疑似泊松/计数 |
| C-连续大方差 | f_19~f_26 (8列) | float64 | std≈2.3~2.5，可能是 A×B 的交互衍生 |
| 特殊-字符串 | f_27 | str | 固定10字符 A-Z，高基数，需拆分 |
| 特殊-连续 | f_28 | float64 | std≈238，异常大，疑似多特征乘积 |
| 特殊-二值 | f_29 | int64 | 0/1，mean=0.35 |
| 特殊-三值 | f_30 | int64 | 0/1/2，mean=0.98 |

## 关键发现

1. **特征交互是竞赛核心**：官方描述明确提到 "various feature interactions"，三组特征结构暗示 C 组可能由 A×B 交互生成，需逆向工程这种关系
2. **f_27 是隐藏信息载体**：10位定长字符串，每位 A-Z，应拆分为 10 个独立类别特征（f_27_0 ~ f_27_9），可能编码了高阶类别交互
3. **f_28 量级异常**：值域 [-851, 891]，std≈238，远超其他特征，需要特殊处理（log变换/分箱/rank变换）
4. **无数据质量问题**：无缺失值、目标平衡、无需采样策略，可直接建模

## 建模建议

### 推荐方法（优先级排序）
1. **LightGBM**（主力）：n_jobs=16，原生处理类别特征，快速迭代
2. **XGBoost**（次选）：n_jobs=16，与 LGB 集成可提升稳定性
3. **CatBoost**（辅助）：天然处理类别特征，对 f_27 拆分后的字符特征友好
4. **最终方案**：加权集成 / Stacking

### 特征工程方向
| 优先级 | 方向 | 具体操作 |
|--------|------|----------|
| P0 | f_27 拆分 | 拆为 10 个单字符类别特征，label encode |
| P0 | 组间交互 | A组 × B组 两两交互（如 f_00 * f_07） |
| P1 | f_28 变换 | rank 变换 / 分箱 / np.sign(x)*np.log1p(abs(x)) |
| P1 | B组聚合 | f_07~f_18 求和/均值/最大值作为新特征 |
| P2 | A组多项式 | A组内两两乘积 |
| P2 | 频率编码 | 对 f_27 各位字符做频率编码 |

### 评估策略
- **CV**：StratifiedKFold, k=5
- **metric**：AUC（注意提交的是概率值，非类别标签）
- **风险**：800K 样本 + 交互特征爆炸可能导致内存压力，需控制特征数量

### 预期难度
- **中等**：数据干净、目标平衡、GBDT 友好，但竞赛区分度在特征交互工程的深度上

## 模板选择理由

选择 **`tabular`** 模板：

| 判据 | 匹配情况 |
|------|----------|
| 结构化表格数据 | ✅ 纯表格，无图像/文本/序列 |
| GBDT 为主力模型 | ✅ LightGBM/XGBoost/CatBoost |
| 标准 train/test split | ✅ train.csv + test.csv |
| 标准提交格式 | ✅ id + target 概率值 |

**各 slot 初步方向：**

| Slot | 实现方向 |
|------|----------|
| `preprocess` | f_27 拆分 → label encode；f_28 rank 变换；B组 int→category |
| `feature_engineer` | 组间交互（A×B）；B组聚合统计；f_27 字符频率编码 |
| `model_train` | LightGBM 5-fold CV，early_stopping，categorical_feature 指定 |
| `model_predict` | predict_proba[:, 1] 输出概率 |
| `ensemble` | LGB + XGB + CatBoost 加权平均（权重由 CV AUC 决定） |
| `postprocess` | clip(0, 1) 确保概率合法；生成 submission.csv |
