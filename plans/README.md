# 执行计划目录

> 计划是一等公民。所有开发任务和实验任务都通过本目录管理。

## 目录结构

```
plans/
├── README.md           # 本文件
├── active/             # 当前活跃计划
├── completed/          # 已完成计划归档
└── tech_debt.md        # 技术债务追踪
```

## 计划文件命名规范

- 格式：`{序号}_{简短描述}.md`
- 示例：`001_implement_draft_pes.md`
- 序号：三位数字，从 001 开始递增

## 计划文件模板

```markdown
# [计划标题]

## 元信息
- 状态: draft / in_progress / blocked / completed
- 创建: YYYY-MM-DD
- 负责人: [Agent/人类]

## 目标
[1-3 句话描述目标]

## 检查点
- [ ] [检查点 1]
- [ ] [检查点 2]

## 阻塞项
- [阻塞项描述] — [等待什么]

## 决策日志
- YYYY-MM-DD: [决策] — [理由]
```

## 状态定义

| 状态 | 含义 |
|------|------|
| draft | 计划编写中，未开始执行 |
| in_progress | 正在执行 |
| blocked | 阻塞中，等待外部条件 |
| completed | 已完成 |

## 工作流程

1. 在 `active/` 创建计划文件
2. 执行计划，更新状态和检查点
3. 完成后移动到 `completed/`
